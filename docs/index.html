What is Timelapse?

Timelapse is a tool suite that simplifies capturing and investigating behavior in web applications. Timelapse captures interactions with a web application and creates an exact execution recording. A debugging tool can visualize and re-execute the captured interactions at will. Developers can use standard debugging tools on the re-executing program to better understand what happened. This tool implemented as an extension to the the Web Inspector suite (used by Chrome and Safari).

Timelapse is based on deterministic record/replay technology from the virtual machine literature. It is also a foundation for research into topics such as omniscient debugging, interactive code coverage tools, and benchmark/test synthesis. For more details on the research behind Timelapse, see the Research page.



Timelapse is an open source research project. It is based on current and past research conducted at CSE (University of Washington) and at S3 (Purdue University).

The project leader is Brian Burg (@brrian). However, contributions are welcome from anyone.

Supported platforms

Timelapse is currently designed to work with the Mac Safari browser. Primary development of Timelapse takes place on the Darwin/OS X port of WebKit. Details of cross-platform support are on the FAQ page.

Collaboration and bug reports

If you are interested in collaborating, or are considering using Timelapse for your research but have questions, feel free to send an email.

A paper describing Timelapse's user interface, infrastructure, evaluation, and a user study is in submission. The paper will be available shortly. Our user study materials are freely available.

For UW CSE and research collaborators, ask to be added to the timelapse developers mailing list (timelapse@cs). If you would like to contact Brian, the primary developer, his email is burg@cs.

Reporting bugs is best done on bitbucket or by direct email. The FAQ page has more information on bug reporting and ways that you can contribute.

Implementation

The proof-of concept implementation of Timelapse is an unofficial, experimental fork of WebKit. Accordingly, most of the source is licensed under the same terms as WebKit itself (parts LGPL, parts BSD). New code is typically under a BSD-style license.

We use Mercurial for version control. The public repository is hosted on bitbucket.org under burg/timelapse.

We do not publicly post Timelapse-enabled versions of Safari at this time. Prebuilt binaries are available on request. To obtain and build Timelapse yourself, please consult the documentation pages for repositories and building.
